**Checkout route** (`src/app/api/billing/checkout/route.ts`):
POST handler. Authenticates user via `getUser()`. Looks up `stripe_customer_id` from profiles table. If no customer, creates one via `stripe.customers.create({ email: user.email })` and saves to profiles. Creates Checkout session:
```typescript
const session = await stripe.checkout.sessions.create({
  customer: customerId,
  mode: 'subscription',
  line_items: [{ price: process.env.STRIPE_PRICE_ID!, quantity: 1 }],
  subscription_data: {
    trial_period_days: 14,
    trial_settings: {
      end_behavior: { missing_payment_method: 'cancel' },
    },
  },
  payment_method_collection: 'if_required',
  success_url: `${process.env.NEXT_PUBLIC_URL}/account?success=true`,
  cancel_url: `${process.env.NEXT_PUBLIC_URL}/account?canceled=true`,
})
```
Returns `{ url: session.url }`. Uses `createAdminClient()` for DB writes (bypass RLS for customer ID save).

CRITICAL: `payment_method_collection: 'if_required'` ensures no credit card during trial (BILLING-01). `trial_settings.end_behavior.missing_payment_method: 'cancel'` auto-cancels when trial ends without card.

**Webhook handler** (`src/app/api/webhooks/stripe/route.ts`):
POST handler. Reads raw body via `request.text()`. Verifies signature with `stripe.webhooks.constructEvent(body, signature, process.env.STRIPE_WEBHOOK_SECRET!)`. Uses `createAdminClient()` (bypasses RLS).

Handle these events:
1. `checkout.session.completed` -- Extract `customer` and `subscription` from event. Update profiles: set `stripe_subscription_id`, `subscription_status = 'trialing'` (or 'active' if no trial).
2. `invoice.paid` -- Extract `customer` from invoice. Update profiles: set `subscription_status = 'active'`, `current_period_end = new Date(invoice.period_end * 1000).toISOString()`.
3. `customer.subscription.updated` -- Extract `customer` and `status` from subscription. Update profiles: set `subscription_status` to mapped value (trialing/active/past_due/canceled).
4. `customer.subscription.deleted` -- Extract `customer`. Update profiles: set `subscription_status = 'canceled'`.
5. `invoice.payment_failed` -- Extract `customer`. Update profiles: set `subscription_status = 'past_due'`.
6. `customer.subscription.trial_will_end` -- Log for now (email notification added later).

All DB updates use `.eq('stripe_customer_id', customerStripeId)` to find the correct profile row.

Return `{ received: true }` with 200 for all handled events. Return 400 only for signature verification failure.

CRITICAL: Do NOT add `export const config = { api: { bodyParser: false } }` -- this is Pages Router syntax. In App Router, `request.text()` already gives the raw body.

**Subscription check utility** (`src/lib/billing/subscription-check.ts`):
```typescript
import { createClient } from '@/lib/supabase/server'

export type SubscriptionStatus = 'trialing' | 'active' | 'past_due' | 'canceled' | 'none'

export interface SubscriptionInfo {
  status: SubscriptionStatus
  trialEndsAt: string | null
  currentPeriodEnd: string | null
  isActive: boolean  // true for 'trialing' or 'active'
}

export async function checkSubscription(userId: string): Promise<SubscriptionInfo> {
  const supabase = await createClient()
  const { data } = await supabase
    .from('profiles')
    .select('subscription_status, trial_ends_at, current_period_end')
    .eq('id', userId)
    .single()

  if (!data) return { status: 'none', trialEndsAt: null, currentPeriodEnd: null, isActive: false }

  const status = (data.subscription_status as SubscriptionStatus) || 'none'
  return {
    status,
    trialEndsAt: data.trial_ends_at,
    currentPeriodEnd: data.current_period_end,
    isActive: status === 'trialing' || status === 'active',
  }
}

export function isSubscriptionActive(status: SubscriptionStatus): boolean {
  return status === 'trialing' || status === 'active'
}
```
