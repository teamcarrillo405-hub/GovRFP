## Layout Contract

### Editor layout with sidebar

The ProposalEditor currently renders: `<div className="flex gap-0">` with editor column (`flex-1`) + CompliancePanel (`w-80 shrink-0`, right side).

Phase 6 inserts RfpStructureSidebar to the **left** of the editor column, inside the same flex container:

```
[RfpStructureSidebar (w-64, collapsible)] [Editor column (flex-1)] [CompliancePanel (w-80)]
```

When sidebar is collapsed, it renders as a `w-10` strip containing only the toggle button — the editor column expands to fill reclaimed space via `flex-1`.

### Sidebar dimensions

| State | Width | Behavior |
|-------|-------|----------|
| Expanded | 256px (`w-64`) | Full sidebar with section list |
| Collapsed | 40px (`w-10`) | Strip with toggle button only; no section content visible |

Transition: `transition-all duration-200 ease-in-out` on width — matches existing ProposalEditor tab `transition-colors` convention. No animation library needed.

### Sidebar internal structure (expanded state)

```
┌─────────────────────────────┐
│ [≡] RFP Structure    [←]   │  ← Header: px-4 py-3, border-b
├─────────────────────────────┤
│ ▶ Section Title         (N) │  ← Row: px-4 py-2, collapsed
│ ▼ Section Title         (N) │  ← Row: px-4 py-2, expanded (active)
│   · Requirement text        │  ← Requirement: px-4 py-1.5 pl-8, text-xs
│   · Requirement text        │
│ ▶ Section Title         (N) │
└─────────────────────────────┘
```

### Active section highlighting (SIDEBAR-06)

When editor scrolls, the sidebar row corresponding to the currently visible proposal section receives:
- Left border: `border-l-2 border-blue-700`
- Background: `bg-white`
- Title: `text-blue-700 font-semibold`

Inactive rows: no left border, `bg-transparent`, `text-gray-700 font-normal`.

## Interaction Contract

### Toggle (SIDEBAR-02)

- Toggle button: SVG chevron `«` (expanded) / `»` (collapsed), 16x16, `text-gray-500 hover:text-gray-700`
- Button placement: top-right of sidebar header when expanded; top of collapsed strip
- Keyboard: `aria-label="Collapse RFP sidebar"` / `aria-label="Expand RFP sidebar"`
- State stored: `useState<boolean>(true)` — sidebar starts expanded

### Section expand/collapse (SIDEBAR-04)

- Chevron icon: right-pointing `▶` when collapsed, down-pointing `▼` when expanded
- SVG inline, 12x12, `text-gray-400`
- Click target: full section row (not just chevron)
- Multiple sections can be expanded simultaneously (no accordion constraint)
- Default state: all sections collapsed — only titles and badges visible

### Click-to-scroll navigation (SIDEBAR-05)

- Click on a section title fires a scroll command to the Tiptap editor
- Mechanism: `editor.commands.scrollIntoView()` after setting selection to the heading node matching the section title
- Section titles in rfp_structure match proposal section names from SECTION_NAMES
- The section row is a `<button>` element, not a link — no URL change

### Active section tracking (SIDEBAR-06)

- Uses Tiptap `onSelectionUpdate` + `onUpdate` callbacks to detect current cursor heading
- Alternatively: `IntersectionObserver` on section heading DOM nodes
- Active section prop passed down from ProposalEditor (or internal sidebar state via callback)
- Highlight updates on scroll without user interaction

### No-data handling (SIDEBAR-01 / SIDEBAR-07)

- If `rfpStructure` is null or has 0 sections: sidebar renders empty state (see Copywriting Contract)
- Sidebar still renders (collapsed) — toggle button always accessible

## Accessibility Contract

- Sidebar uses `role="complementary"` and `aria-label="RFP structure panel"`
- Section list uses `role="list"` with `role="listitem"` per section row
- Expand/collapse buttons include `aria-expanded` attribute (true/false)
- Toggle button is always focusable regardless of sidebar state
- Color is never the sole differentiator for active state — active row also uses `font-semibold` and left border
- Keyboard navigation: Tab moves through section rows; Enter/Space fires click on buttons
