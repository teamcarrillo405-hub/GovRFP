**Placement:** Inside the editor page header area, positioned in the top-right area of the `<main>` container alongside the breadcrumb + proposal title row. Specifically: add a `flex items-center justify-between` wrapper around the existing `<h1>` and place `<ExportButtons>` on the right side.

**Layout:**

Two buttons in `flex gap-2` row. Word button uses accent fill. PDF button uses outline style.

**Button anatomy â€” Word (primary):**
- Label: "Export Word"
- Loading label: "Exporting..."
- Classes: `inline-flex items-center gap-2 px-3 py-2 bg-blue-700 text-white text-sm font-semibold rounded-md hover:bg-blue-800 transition-colors disabled:opacity-50 disabled:cursor-not-allowed`
- SVG icon: 16Ă—16 document-with-arrow-down icon, `aria-hidden="true"`

**Button anatomy â€” PDF (secondary):**
- Label: "Export PDF"
- Loading label: "Exporting..."
- Classes: `inline-flex items-center gap-2 px-3 py-2 border border-gray-300 text-gray-700 text-sm font-semibold rounded-md hover:bg-gray-50 bg-white transition-colors disabled:opacity-50 disabled:cursor-not-allowed`
- SVG icon: 16Ă—16 document-with-PDF badge icon, `aria-hidden="true"`

**States:**
1. Idle â€” both buttons enabled, full color
2. Downloading Word â€” Word button shows "Exporting..." with spinner SVG (`animate-spin h-3 w-3`), PDF button remains enabled
3. Downloading PDF â€” PDF button shows "Exporting..." with spinner SVG, Word button remains enabled

## Interaction Contract

### Export flow (happy path)

1. User clicks "Export Word" or "Export PDF"
2. Button immediately switches to "Exporting..." with spinner â€” user gets instant feedback that the request is in flight
3. Browser triggers file download via `URL.createObjectURL` + dynamic `<a>` click â€” no page navigation
4. Button returns to idle state
5. File appears in browser's download bar

### Export flow (error path)

1. User clicks export
2. Button shows "Exporting..." with spinner
3. API returns non-2xx response
4. Button returns to idle
5. Error message appears inline below the button group: "Export failed â€” try again" in `text-xs text-red-600`
6. Error clears after 5 seconds (`setTimeout` with cleanup) or on next export attempt

### Loading state rules

- Word and PDF downloads are independent â€” each has its own loading boolean
- Downloading Word does NOT disable the PDF button (and vice versa)
- Both buttons CAN be active simultaneously if user clicks both quickly (each has its own fetch lifecycle)

### Placement in editor page

The editor page (`src/app/(dashboard)/proposals/[id]/editor/page.tsx`) currently has:
```
<h1 className="text-xl font-semibold text-gray-900 mb-6">{proposal.title}</h1>
<ProposalEditor ... />
```

Phase 5 changes this to:
```
<div className="flex items-center justify-between mb-6">
  <h1 className="text-xl font-semibold text-gray-900">{proposal.title}</h1>
  <ExportButtons proposalId={id} />
</div>
<ProposalEditor ... />
```

`ExportButtons` is a `'use client'` component imported into the Server Component page â€” same pattern as `ProcessingStatus`, `FileUpload`, and `ProposalEditor`.
