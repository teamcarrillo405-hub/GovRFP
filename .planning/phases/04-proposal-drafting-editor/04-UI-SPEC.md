| Token | Value | Usage |
|-------|-------|-------|

## Interaction States

### Streaming State

Triggered: user clicks Generate or Regenerate, SSE stream opens.

1. Generate button becomes disabled (`opacity-50 cursor-not-allowed`)
2. Editor canvas shows `bg-amber-50 border-amber-200` border (streaming active color)
3. Streaming overlay appears on canvas with "Generating..." text
4. Section tab for active section shows `bg-blue-50 text-blue-700` while streaming
5. Auto-save indicator shows "Generating..." — auto-save is suspended during stream
6. On stream complete: overlay removes, canvas border returns to `border-gray-200`, auto-save resumes

### Auto-save State Machine

```
idle → saving (every 30s or on tab switch) → saved | error
saved → idle (timestamp shown)
error → idle (retry on next interval)
```

- Auto-save interval: 30 seconds (EDITOR-02)
- Save also triggers on: section tab change, RegenerateDialog confirm
- Timestamp format: "Saved at HH:MM:SS" (24-hour, local time, `toLocaleTimeString()`)

### Compliance Gap Highlighting (EDITOR-04)

- Applied: after each auto-save, compliance-scanner runs on current section JSON
- Gap mark: `bg-amber-100` background applied via ComplianceGap Tiptap Mark extension
- Marks apply to the entire paragraph containing insufficient keyword coverage — not word-level
- Marks are stripped before Tiptap JSON is stored to Supabase (stored clean; marks re-applied on load)
- Marks are invisible in exported Phase 5 output (export reads clean JSON)

### Section Tab Switching

- Switching tabs auto-saves current section before switching
- If streaming is active: tab switch is disabled (tabs unclickable during generation)
- Tab disabled state: `pointer-events-none opacity-60`

## Implementation Notes for Executor

1. **No shadcn gate needed.** This project deliberately uses Tailwind-only without shadcn. Do not initialize shadcn during Phase 4.

2. **Tiptap client boundary.** `useEditor` must only run inside `'use client'` components. `SectionEditor`, `EditorToolbar`, `ProposalEditor`, `CompliancePanel`, and `RegenerateDialog` are all client components. The page at `proposals/[id]/editor/page.tsx` is a server component that passes data as props.

3. **ComplianceGap mark exclusion.** When saving Tiptap JSON to Supabase, strip ComplianceGap marks before the upsert. Re-apply marks client-side after load by running the compliance scanner against the loaded JSON.

4. **Streaming buffer pattern.** Do not call `editor.commands.insertContent()` per SSE chunk. Buffer the full streamed text string, then call `editor.commands.setContent(bufferedHtml)` once the stream closes (or use `editor.commands.setContent(json)` with constructed Tiptap JSON). See RESEARCH.md Pattern 2 for the route handler.

5. **Auto-save suspension.** Clear and reset the auto-save interval when streaming starts. Do not attempt to save while Claude is writing into the editor.

6. **Compliance panel re-render.** `CompliancePanel` receives `sections` state as prop from `ProposalEditor`. It re-renders automatically after each auto-save when the parent state updates.

7. **Font.** Geist Sans is already loaded in globals.css via `--font-geist-sans`. The editor canvas inherits this. No additional font loading needed.

8. **No emojis anywhere.** Use inline SVG icons for all toolbar controls and status indicators. The MEMORY rule is explicit: no emojis, no emoticons — cheap. Use SVG icons, color, or typography instead.
