3. Create `src/components/editor/ProposalEditor.tsx` ('use client'):
   - Props: `{ proposalId: string, initialSections: ProposalSection[], requirements: AnalysisRequirement[], complianceMatrix: ComplianceMatrixRow[] }`
   - State: `activeSection` (SectionName, default 'Executive Summary'), `sections` (Map of SectionName -> { content, draftStatus, lastSavedAt }), `isStreaming` (boolean), `streamBuffer` (string), `complianceCoverage` (ComplianceCoverage per section)
   - Initialize sections state from initialSections prop
   - **Section tabs**: horizontal tab bar per UI-SPEC. Active tab: `border-b-2 border-blue-700 text-blue-700 bg-white`. Inactive: `text-gray-500 hover:text-gray-700 bg-gray-50 hover:bg-gray-100`. Tab item: `px-4 py-2 text-sm font-medium cursor-pointer transition-colors`. When streaming, tabs have `pointer-events-none opacity-60`.
   - **Two-column layout**: `<div className="flex gap-0">` — editor column `flex-1 min-w-0` + CompliancePanel `w-80 shrink-0`
   - **Streaming handler** (`handleGenerate(section, instruction?)`):
     a. Set `isStreaming = true`, `streamBuffer = ''`
     b. Fetch POST `/api/proposals/${proposalId}/draft` with `{ section, instruction }`
     c. Read response.body with `getReader()` + `TextDecoder`
     d. Parse SSE lines: filter `data: ` prefix, parse JSON, extract `content_block_delta` / `text_delta` events
     e. Accumulate into `fullText` string, update `streamBuffer` state for preview
     f. On stream end: call `editorRef.current?.commands.setContent(fullText)` ONCE (buffer pattern from RESEARCH.md)
     g. Set `isStreaming = false`, mark section as `draft_status: 'draft'`
   - **Auto-save logic** (30-second interval):
     a. `useRef` for `saveTimerRef`, `isDirtyRef`, `isSavingRef`, `isStreamingRef`
     b. `useEffect` sets up `setInterval(30_000)` that:
        - Checks `!isDirtyRef.current || isSavingRef.current || isStreamingRef.current` — skip if any
        - Gets current editor JSON via ref
        - Strips compliance marks via `stripComplianceMarks(json)`
        - Calls PATCH `/api/proposals/${proposalId}/sections` with `{ section_name, content: cleanJson, draft_status: 'edited' }`
        - On success: update `lastSavedAt` state, reset `isDirtyRef`
        - On error: set save status to 'error'
     c. Auto-save also triggers on section tab switch (save current before switching)
     d. Clear interval on unmount
   - **Compliance scanning**: after each successful save, run `scanCompliance(sectionJson, requirements, activeSection)` and update `complianceCoverage` state
   - **Apply ComplianceGap marks**: after scan, for each 'unaddressed' requirement, find paragraph nodes in editor that should be marked and apply `setMark('complianceGap', { requirementId })` via editor chain
   - Render: section tabs, then `<EditorToolbar editor={editorRef.current} />`, then `<SectionEditor>` canvas, then GenerateBar with auto-save indicator + generate/regenerate button
